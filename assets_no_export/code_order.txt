Here is following good practise of code order in GDScript:
	01.	@tool, @icon, @static_unload, @abstract
	02.	class_name
	03.	extends
	04.	## Short class doc description
		## 
		## Long class doc description
	
	05.	signals
	06.	enums
	07.	constants
	08.	static variables
	09.	@export variables
	10.	remaining regular variables
	11.	@onready variables
	
	12.	_static_init()
	13.	remaining static methods:
		1.	public static methods
		2.	private static methods
	14.	overriden built-in virutal methods:
		1.	_init()
		2.	_enter_tree()
		3.	_ready()
		4.	_process()
		5.	_physics_process()
		6.	remaining virtual methods
	15.	overriden custom methods:
		1.	public custom methods
		2.	private custom methods
	16.	remaining methods:
		1.	remaining public methods
		2.	remaining private methods
	17.	inner classes
